# Target Users & Use Cases

## Primary User

Linux (and macOS) power users who want professional VJ software without being locked into Windows or macOS. Technical people — comfortable with shaders, command line, open source tooling — but who need a **production-ready** tool, not a toy.

## Use Cases (All Must Be Supported)

### 1. Live Club/Festival Performance
- VJ controls visuals in real-time via MIDI controller (Akai APC Mini, etc.)
- A/B channel crossfading between deck groups
- Audio reactivity tied to the house sound system
- Quick scene switching, fast parameter tweaking
- Rock-solid stability — cannot crash mid-set
- 60fps minimum at output resolution

### 2. Art Installations (Unattended)
- Preset transitions and automations run without human input
- Auto-transitions between decks on a timer or trigger
- Long-running stability (hours/days without intervention)
- Generative shaders that evolve over time
- Possible multi-output to different surfaces

### 3. Corporate / Event Production
- Projection mapping onto surfaces (basic quad warping at minimum)
- Multi-output to multiple projectors/displays
- Scene presets loaded and triggered on cue
- Professional output quality (1080p minimum, 4K goal)

### 4. Creative Development / Shader Authoring
- Write ISF/GLSL shaders and see them live with hot-reload
- Test shaders with audio input, parameter tweaking
- Export/share shaders in standard ISF format
- Not the primary use case but should be a natural byproduct

## What This Is NOT

- Not a creative coding environment (use TouchDesigner/Processing for that)
- Not a video editor (no timeline)
- Not a 3D engine (shaders operate on 2D textures)
- Not Windows-first (Linux and macOS are the targets)
